Installed. Here is what to do next

haven-agentd is a background tool with no window. Open Terminal and run these three steps. Each prints a clear status report.

  1. Set it up (creates config + background service, generates a local token): 
    haven-agentd setup \
  --domain staging.haven.digipomps.org \
  --resolver-url https://staging.haven.digipomps.org \
  --sprout-path /usr/local/libexec/havenagent/sprout
  2. Request provisioning — send the output to your operator, who returns a pack.json, then import it: 
    haven-agentd provisioning-request
haven-agentd provisioning-import --pack ~/Downloads/pack.json
  3. Check readiness at any time: 
    haven-agentd bootstrap-probe

Claude/Codex correspondence pilot: if your operator supplied a single-use haven-invite.json, enroll the messages-only bridge:

haven-correspondence-mcp setup --invite ~/Downloads/haven-invite.json
haven-correspondence-mcp doctor --profile <profile-from-invite>

Full guide: open /usr/local/share/havenagent/QUICKSTART.md. Correspondence guide: open /usr/local/share/havenagent/ASSISTANT_CORRESPONDENCE.md. Until provisioning is imported, the general agent runs local-only.
